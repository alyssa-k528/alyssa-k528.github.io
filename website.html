Packet Transmissions


Worked with socket programming, and converted the connectionless UDP to connection oriented by the use of acknowledgments and created a function that randomly drops packets for both the client and server sides. The client sends requests and both the server and client side have possibilities of dropping the packet. Functions made to deal with dropped packet scenarios as well as packet retransmissions, used timeouts and retranmissions, following the Go-Back-N protocol.
